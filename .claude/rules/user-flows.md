# Пользовательские сценарии

## 🎯 Основные пользовательские потоки

### 1. Создание базовой цветовой схемы

**Цель**: Пользователь хочет сгенерировать цветовую схему Material Design 3 на основе seed color

**Шаги**:
1. Открывает страницу сервиса
2. Выбирает seed color через OKLCH color picker или вводит HEX код
3. Выбирает вариант схемы (Tonal Spot, Vibrant, Expressive, Neutral, Monochrome, Fidelity, Content, Rainbow, Fruit Salad)
4. Выбирает спецификацию Material Design (2025 или 2021)
5. (опционально) Кастомизирует некоторые цвета ролей (primary, secondary, tertiary, error, neutral, neutralVariant)
6. Нажимает "Get JSON" для просмотра результата
7. Изменяет настройки json, если это необходимо
8. Получает полную цветовую схему с светлой и темной темами, копирует или сохраняет JSON

**Результат**: Готовая к использованию цветовая схема Material Design 3 в JSON формате

**Особенности**:
- Реактивная генерация: схема обновляется автоматически при изменении настроек
- OKLCH цветовое пространство для более точного контроля над цветами
- Визуальная индикация sRGB гаммы на слайдерах

### 2. Добавление расширенных цветов

**Цель**: Пользователь хочет добавить дополнительные брендовые или тематические цвета

**Шаги**:
1. После создания базовой схемы нажимает "Add a color"
2. Выбирает цвет через OKLCH color picker или вводит HEX код
3. Указывает название цвета (например, "Brand Blue")
4. При необходимости добавляет еще цвета
5. Схема автоматически перегенерируется с включением новых цветов и их вариантов

**Результат**: Расширенная схема с пользовательскими цветами и их полными тональными палитрами

**Особенности**:
- Автоматическое создание container и on-цветов для каждого расширенного цвета
- Генерация state layers для всех новых цветов
- Интеграция с общей схемой Material Design 3
- Возможность удаления добавленных цветов

### 3. Настройка экспорта JSON

**Цель**: Пользователь хочет получить схему в нужном формате для своего проекта

**Доступные опции**:

#### Форматы именования
- **kebab-case**: `primary-container`, `on-primary-container`
- **camelCase**: `primaryContainer`, `onPrimaryContainer`  
- **Title Case**: `Primary Container`, `On Primary Container`

#### Дополнительные данные
- **State Layers**: включение слоев состояний (hover, focus, pressed, dragged, disabled)
- **Tonal Palettes**: полные тональные палитры (0-100 тонов) для всех цветов
- **W3C Design Tokens Format**: совместимость с Figma Variables и Tokens Studio

#### Collection Name
- Настраиваемое имя коллекции для W3C формата
- По умолчанию: "Semantic colors"

**Шаги**:
1. В панели JSON результатов выбирает нужный формат именования
2. Включает/выключает State Layers и Tonal Palettes
3. При необходимости включает W3C формат и задает название коллекции
4. JSON автоматически переформатируется согласно настройкам

### 4. Экспорт и использование результатов

**Цель**: Сохранить или поделиться сгенерированной схемой

**Варианты экспорта**:

#### 4.1 Копирование в буфер обмена
- Кнопка "Copy" копирует JSON в буфер обмена
- Визуальное подтверждение успешного копирования
- Готово для вставки в IDE или design tools

#### 4.2 Скачивание файла
- Кнопка "Download JSON" сохраняет файл `material-colors.json`
- Готов для импорта в проекты или Figma

## 🔄 Интерактивные возможности

### OKLCH Color Picker

**Особенности**:
- **Только OKLCH цветовое пространство** - без выбора других форматов
- **Dropdown интерфейс** - открывается при клике на цветовой превью
- **Синхронизация в реальном времени** между picker, HEX input и превью
- **Визуальная индикация sRGB гаммы**:
  - Серые полосы для недоступной яркости
  - Красные полосы для пересыщенных цветов
  - Обесцвеченные области для недоступных оттенков
- **Автоматическое закрытие** при клике вне области

### Реактивные обновления

**Триггеры автоматической генерации**:
- Изменение seed color
- Выбор другого стиля генерации
- Переключение спецификации Material Design
- Изменение расширенных цветов
- Изменение названий расширенных цветов

**Триггеры переформатирования JSON**:
- Изменение формата именования
- Включение/выключение State Layers
- Включение/выключение Tonal Palettes  
- Переключение W3C формата
- Изменение названия коллекции

### Валидация в реальном времени

**HEX коды**:
- Проверка корректности формата при вводе
- Синхронизация между OKLCH picker и HEX полем
- Визуальная индикация некорректных значений

**Расширенные цвета**:
- Валидация названий и HEX кодов
- Предотвращение дублирования названий
- Автоматическая санитизация названий для JSON ключей
